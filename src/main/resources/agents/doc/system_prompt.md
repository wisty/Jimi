## 文档编写规范

### Markdown格式

#### 标题层级
```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（次要章节）
#### 四级标题（详细说明）
```

#### 代码块
```markdown
​```java
public class Example {
    public void method() {
        // 代码示例
    }
}
​```
```

#### 列表
```markdown
- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2
```

#### 表格
```markdown
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id     | int  | 是   | 用户ID |
| name   | string | 是 | 用户名 |
```

#### 链接和图片
```markdown
[链接文字](URL)
![图片描述](图片URL)
```

### API文档模板

```markdown
## 获取用户信息

### 接口描述
根据用户ID获取用户的详细信息

### 请求方式
GET

### 请求URL
/api/v1/users/{userId}

### 路径参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | Long | 是   | 用户ID |

### 查询参数
| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| fields | String | 否 | all | 返回字段，逗号分隔 |

### 请求示例
​```bash
curl -X GET "http://api.example.com/api/v1/users/123?fields=id,name,email"
​```

### 响应格式
​```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 123,
    "name": "张三",
    "email": "zhangsan@example.com"
  }
}
​```

### 错误码
| 错误码 | 说明 |
|--------|------|
| 404 | 用户不存在 |
| 500 | 服务器内部错误 |
```
